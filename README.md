# SOZO — путь к свободе от созависимости

Учебный сайт-курс для тех, кто теряет себя в близких отношениях. Последовательный путь из трёх шагов:

1. **Понять** — что такое созависимость, как она проявляется и откуда берётся.
2. **Границы** — практические навыки: где заканчиваюсь я и начинается другой.
3. **Свобода** — корень и исцеление: освобождение через доверие Богу.

Вера — в основе, психологические инструменты (границы, навыки, карта типов) — как практические средства. Вход мягкий, рассчитан и на людей, ещё далёких от церкви. Название — от греческого σώζω («со́зо»): спасать, исцелять, избавлять, делать целым.

Проект построен в той же технологии и стилистике, что и сайт «Основание» общины «Ковчег спасения», но является самостоятельным.

## Как запустить на своём компьютере

Нужен установленный [Node.js](https://nodejs.org) (версии 18 или новее).

```bash
npm install      # один раз — установить зависимости
npm run dev      # запустить сайт в режиме разработки
```

После запуска откройте показанный адрес (обычно http://localhost:5173).

Собрать готовую версию для публикации:

```bash
npm run build    # результат появится в папке dist/
npm run preview  # посмотреть собранную версию локально
```

## Как опубликовать в интернете

Сайт — статический, его можно бесплатно разместить на **Vercel**, **Netlify** или **GitHub Pages**.

- **Vercel / Netlify:** подключите репозиторий — сборка произойдёт автоматически (команда `npm run build`, папка `dist`). В `vite.config.ts` оставьте `base: "/"`.
- **GitHub Pages:** если адрес будет вида `https://<логин>.github.io/<имя-репо>/`, поменяйте в `vite.config.ts` строку на `base: "/<имя-репо>/"`.

После публикации обновите ссылки `og:url` и `og:image` в `index.html` на ваш настоящий адрес — это карточка, которая показывается при пересылке ссылки в мессенджерах.

## Как устроен контент (и как добавлять уроки)

Все уроки лежат в папке `src/content`. Структура — это вложенные папки: **шаг → модуль → урок**. Числовой префикс (`01-`, `02-`…) задаёт порядок и в адресе не показывается.

```
src/content/
  01-ponyat/                     ← Шаг
    _meta.json                   ← название и описание шага
    01-chto-takoe/               ← Модуль
      _meta.json                 ← название и описание модуля
      01-kogda-lyubov...md       ← Урок (текст)
      01-kogda-lyubov...quiz.json        ← тест к уроку (необязательно)
      03-otkuda...reflection.json        ← размышление к уроку (необязательно)
```

### Добавить новый урок

1. Создайте в нужном модуле файл `NN-nazvanie.md`, где `NN` — порядковый номер (например `04-...`).
2. Первая строка файла — заголовок урока через `# ` (он же станет названием в списке).
3. При желании добавьте рядом файл с тем же именем и одним из окончаний:
   - `…quiz.json` — тест с автопроверкой;
   - `…reflection.json` — задание-размышление с самопроверкой по эталону.
4. Урок без теста и без размышления засчитывается как пройденный сразу при открытии.

Уроки открываются **строго по очереди**: следующий доступен только после того, как пройден предыдущий.

### Формат текста урока (Markdown)

Обычный Markdown. Для цитат из Писания используйте формат:

```markdown
> Познаете истину, и истина сделает вас свободными.
> <cite>Иоанна 8:32</cite>
```

Для деликатной врезки (например, о безопасности) — блок:

```markdown
<div class="callout">
Текст важной заметки.
</div>
```

### Формат теста (`…quiz.json`)

```json
{
  "passing_score": 75,
  "questions": [
    { "type": "single",  "q": "Вопрос?", "options": ["А","Б","В"], "answer": 1, "explanation": "Пояснение." },
    { "type": "multiple","q": "Выберите всё верное", "options": ["А","Б","В"], "answer": [0,2], "explanation": "…" },
    { "type": "boolean", "q": "Утверждение верно?", "answer": true, "explanation": "…" }
  ]
}
```

- `answer` для `single` — номер варианта (с нуля), для `multiple` — массив номеров, для `boolean` — `true`/`false`.
- ⚠️ Внутри текста **не используйте прямые кавычки** `"` — берите русские «ёлочки». Прямая кавычка внутри строки ломает JSON.

### Формат размышления (`…reflection.json`)

```json
{
  "intro": "Необязательное вступление.",
  "prompts": [
    {
      "q": "Вопрос для размышления.",
      "placeholder": "Подсказка в поле ввода…",
      "model": "Эталонный разбор (Markdown), открывается после ответа.",
      "rubric": ["Критерий самопроверки 1", "Критерий 2"]
    }
  ]
}
```

Ответы пользователя на размышления хранятся только в его браузере и никуда не отправляются.

## Настройки

### Контакт наставника — `src/lib/mentorConfig.ts`

Кнопка «Спросить наставника» под размышлениями открывает у пользователя письмо или копирует текст. Укажите свой email и при необходимости отключите кнопку (`enabled: false`).

### Синхронизация прогресса — `src/lib/supabaseConfig.ts`

По умолчанию **выключена** (`enabled: false`): сайт полностью работает и так, прогресс хранится в браузере пользователя.

Чтобы прогресс сохранялся в аккаунте и переносился между устройствами:

1. Создайте бесплатный проект на [supabase.com](https://supabase.com).
2. Создайте таблицу `user_progress` с колонками `user_id` (uuid), `data` (jsonb), `updated_at` (timestamptz) и включите вход по email (magic link).
3. В `supabaseConfig.ts` поставьте `enabled: true` и впишите `url` и публичный `publishableKey` проекта.

> Вставляйте только **публичный** (publishable/anon) ключ — он и предназначен для кода сайта. Секретный (service_role) ключ в код помещать нельзя.

## Технологии

React + TypeScript + Vite. Маршрутизация — `react-router-dom` (HashRouter, удобно для статического хостинга). Markdown — `marked`. Необязательная синхронизация — Supabase. Шрифты — Cormorant Garamond и Spectral.

Материалы курса подготовлены для общины «Ковчег спасения» (г. Житковичи). Содержание опирается на классические и современные наработки по теме созависимости (в т.ч. модель границ Г. Клауда и Дж. Таунсенда, материалы сообществ восстановления) и на библейское основание.
